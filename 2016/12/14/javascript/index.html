  1. 面向对象的编程方法

  • console.log([1, 2, 3] === [1, 2, 3]) // false

    每个新创建的对象都会被保存在内存之中，即时两个创建的对象值相等，她们的引用地址也不同，所以输出false

  • 基本包装对象 string number boolean

  • toString方法的位置

    系统对象（Array）的toString方法存在于系统对象的原型对象之中，自定义对象的toString方法存在于Object之中

  • 创建对象的模式总结

    1. 工厂模式

      产生背景：ECMAScript中无法创建类，聪明的开发人员便发明了一种函数，用函数来封装以特定接口创建对象的细节

> 模式局限性：解决了创建多个相似对象的问题，但却没有解决对象识别的问题( 即怎么知道一个对象的类型 )
，就是在这样的背景下，构造函数模式出现了

2. 构造函数模式




> 构造函数的局限性：每个方法都要在每一个实例上重新创建一遍

解决方法：this对象的存在，我们不用在执行代码之前就把函数绑定到特定的对象上面




> 这种思路虽然解决了两个函数做同一件事的问题，可是新问题又来了：在全局作用于中定义的函数实际上只能被某一个对象调用，这让全局作用域有点名不副实。更让人无法接受的是：如果对象需要定义许多方法，那么就要定义很多全局函数，于是我们自定义的引用类型便没有任何封装性可言了。 **于是便有了原型模式**

3. 原型模式

> 定义：我们创建的每一个函数都有一个 prototype（原型）属性，这个属性是一个指针，指向一个对象，而这个对象的用途是包含可以由特定类型的所有实例共享的属性和方法
